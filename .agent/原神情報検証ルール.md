# 原神情報作成・検証ルール

## 🎯 基本原則（絶対遵守）

### 1. 推測の完全禁止

- 固有名詞（人名、地名、アイテム名等）は**絶対に推測しない**
- 数値データも**絶対に推測しない**
- わからない場合は**必ず調査してから記載**

### 2. 複数ソースでのクロスチェック

- 重要な情報は**最低2つ以上**の信頼できる情報源で確認
- 情報源間で矛盾がある場合は、より信頼性の高いソースを優先
- 可能であれば公式情報を最優先

### 3. 最新情報の確認

- バージョン情報を記載する場合は、現在のバージョンを明記
- 過去のバージョンで変更された可能性がある情報は特に注意
- 「現在」「最新」などの曖昧な表現は避け、具体的な日付やバージョンを記載

## 🔍 信頼できる情報源のレベル分け

### レベル1（最も信頼できる）

- 公式サイト・公式Wiki
- 公式フォーラム（HoYoLAB）の公式発表
- ゲーム内のヘルプ・説明文

### レベル2（信頼できる）

- Genshin Impact Wiki (Fandom)
- GameWith、Altema、Kamigame等の大手日本語攻略サイト
- 海外の主要攻略サイト（IGN、Polygon等）

### レベル3（参考程度）

- 個人ブログ
- YouTube動画（検証系）
- Reddit等のコミュニティ情報

**原則**: レベル1の情報源がある場合は優先。レベル2以上で複数確認を推奨。

## ⚠️ よくある誤りのパターン

### 1. 似た名前の混同

- 例: 「北陸原型」vs「北陸法器の原型」
- 対策: 正式名称をフルで確認

### 2. NPC名の誤記

- 例: 「九条孝行」（存在しない）vs「鯨井椛」（正しい）
- 対策: 必ずwikiや攻略サイトで確認

### 3. 数値の誤り

- 例: 素材数量の間違い、ステータス値の誤記
- 対策: 複数ソースで数値を確認

### 4. 魚・素材名の創作

- 例: 存在しない魚の名前を推測で書く
- 対策: 実在することを確認してから記載

### 5. 入手方法の不正確さ

- 例: 「イベント報酬」だけで具体的な任務名がない
- 対策: できる限り具体的に記載

## 📝 情報作成の手順（推奨）

1. **調査フェーズ**
   - 複数の情報源で基本情報を収集
   - 矛盾する情報があれば追加調査
   - わからない情報は「不明」として残す（推測しない）

2. **検証フェーズ**
   - 該当する検証チェックリストを使用
   - すべての項目を1つずつ確認
   - 確認できない項目は記載しない

3. **記述フェーズ**
   - 確認済みの情報のみを記載
   - 正式名称を使用（略称は避ける）
   - 数値は正確に記載

4. **レビューフェーズ**
   - 作成した内容を再度チェック
   - 固有名詞が正しいか確認
   - 数値が正確か確認

## 🚨 エラー発見時の対応プロトコル

1. **即座の修正**
   - エラーを発見したら直ちに修正
   - 修正前に正しい情報を確認

2. **影響範囲の確認**
   - 同じエラーが他にないか全体を確認
   - 類似の情報も再チェック

3. **記録と共有**
   - 修正内容をコミットメッセージに明記
   - 今後同じ誤りを防ぐためのメモを残す

4. **ルールの更新**
   - 新しいタイプのエラーがあれば、このルールに追記
   - チェックリストに項目を追加

## 💡 情報の信頼性を高めるコツ

- 「～だと思われる」「おそらく～」などの推測表現を避ける
- 情報源が1つしかない場合は、その旨を注記
- バージョン番号や確認日時を記載
- スクリーンショットやゲーム内画像で裏付けがあると理想的
